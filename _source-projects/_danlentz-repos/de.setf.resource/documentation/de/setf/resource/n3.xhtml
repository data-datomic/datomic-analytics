The library provides i/o access to RDF streams encoded as notation3. The operator n3:read reads statements from the stream and n3:format writes statements to a stream. While notation3 is the only directly implemented encoding, others are available by wrapping a stream of the alternative encoding in a respective filter. In the case of file streams, this is accomplished by piping the data through raptor[1]. http locations are NYI. The control structure presumes that the encoding is to be infered from the location. Whice is customary for pathnames, but can be problematic for http sources. In that case, the decision must combine the URI with the content type from a response header in order to determine the value. --- [1]: http://librdf.org/raptor/rapper.html
